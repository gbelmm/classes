Exercise I: Event Data Modeling

  • Find someone in the class to pair with you.
  • Pick an example of an event in your website or application that you would like to record. Examples: signups, purchases, clicks, shares, content creations, comments.
  • Create a data model for that event. Write out a list of properties you would record about that event.
  • Think of two ways you could use this event data to improve your app.
  • We will select a few examples from the class to talk through

    Exercise II: Analyzing Event Data from the class

    1. Click through this website. Your actions on this site will populate our event database.
    2. Take a 5min break to grab some water, get a snack, or go to the restroom. When we get back everyone should have finished the survey.
    The event data for the survey site is organized into these collections:
    • hunk_selections : this event is stored every time someone submits their hunk choice. It also captures event properties like their name and browser information.
    • class_ratings : this event is stored every time someone submits their survey choice for how much they liked the class. Includes several other properties which have been collected during the user session.
    Next Steps:
    • Go here to download the event data.
    • Open the data file in Excel, Numbers, or Google Docs. If you can't view the data, pair with someone who can.
    • See if you can answer the following questions:
      1. How many events are in the hottie_selections collection?
      2. How many unique users submitted hottie selections?
      3. Who was more popular: Slater or Zack?
      4. What was the average rating for the course?
      5. Did people who liked Slater rate the class higher or lower than people who like Zack?
      6. What browser was the most popular in the class?
